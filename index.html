Abstract

The visually impaired rely heavily on hearing and touching (with their cane) to navigate through life. These senses cannot make up for the loss of vision when identifying objects in the user's path. In this paper, we propose NavSense, an assistive device that supplements existing technology to improve navigation and peace of mind in day to day life. NavSense relies on range detection, computer vision, and hardware acceleration mechanisms to provide real-time object identification and context to the user through auditory feedback. In particular, we use four hardware platforms -- Raspberry Pi 3 B+, Coral Accelerator, Coral Development Board, and Intel Neural Computer Stick -- to compare the efficiency of object detection in terms of time and energy during setup and inference phases. Based on these results, it is possible to tailor the design for specific energy-accuracy requirements. Also, we have implemented and used NavSense in real-world scenarios to show its effectiveness.

Bug Reporting System

Design Report   GitHub Link

Worked together with a team to create a bug tracking system for the Santa Clara University(SCU) IT Department. This system allows clients to report bugs in SCU related online applications and check the status of these bugs at a later time. Once submited, a manager assigns the bug to a tester. The tester can add a description, choose a severity, and determine whether the bug is an issue or not. If the tester determines that the bug is an issue, the bug is sent back to the manager to assign a developer to work on a solution. Once a solution is found, the developer can add a description and send the bug back to the original tester to determine if the solution is correct. If the solution is confirmed to work by the tester, the bug exits the development cycle. The manager can view all current and past bugs at any time. The website is programmed in html, javascript, and css while the database is programmed in python using AWS Lambda, AWS API Gateway, and AWS DynamoDB. Personal Contribution: Write the backend API calls and helped interface them with the frontend. Helped create and edit the report.

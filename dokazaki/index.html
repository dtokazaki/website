Projects

Bug Reporting System

Design Report   Website Link   GitHub Link

Worked together with a team to create a bug tracking system for the Santa Clara University(SCU) IT Department. This system allows clients to report bugs in SCU related online applications and check the status of these bugs at a later time. Once submited, a manager assigns the bug to a tester. The tester can add a description, choose a severity, and determine whether the bug is an issue or not. If the tester determines that the bug is an issue, the bug is sent back to the manager to assign a developer to work on a solution. Once a solution is found, the developer can add a description and send the bug back to the original tester to determine if the solution is correct. If the solution is confirmed to work by the tester, the bug exits the development cycle. The manager can view all current and past bugs at any time. The website is programmed in html, javascript, and css while the database is programmed in python using AWS Lambda, AWS API Gateway, and AWS DynamoDB.
